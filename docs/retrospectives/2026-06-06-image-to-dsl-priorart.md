# 1. The fundamental approach — Confirmed, with one novel wrinkle

**Confirmed.** Recovering a generative program from a shape/image is a recognized
inverse problem, formally posed as program synthesis over a grammar. CSGNet
"takes as input a 2D or 3D shape and outputs a CSG program that models it";
InverseCSG: *"Observing that CSG is a formal grammar, we formulate this inverse
CSG problem as a program synthesis problem."* The loop shape we use — render,
score structural similarity, refine — is the established mechanism, **not** pure
feed-forward inference: CSGNet uses a CSG rendering engine in the loop with a
Chamfer-distance reward and "converges … in about 10 iterations"; VIGA (2026)
runs "analysis-by-synthesis as an execution-grounded closed loop" that "writes
candidate programs, executes them to render images, … and iteratively updates
the code." Two competing claims were **refuted** in verification: that the
general inverse problem is flatly "solvable," and that round-trip acceptance is
robust to grammar-inexpressible inputs in general — so do **not** overstate
solvedness (see §6).

**The one wrinkle that is ours alone (Open-in-field).** *No surveyed system reads
producer-embedded authoring-time metadata as ground truth.* Every published loop
(CSGNet, VIGA, InverseCSG) compares the *rendered output* to the target —
perceptually (VIGA's VLM inspection) or structurally (Chamfer/silhouette) — and
none has a producer-of-record whose metadata it trusts. Our **DSL-as-source-of-
truth** (Tenet 23: read `data-sides`/`data-face-class`/`data-symmetry-fold`
instead of re-deriving from raster) is an *engineering shortcut the literature
neither validates nor contradicts*. It is sound precisely *because* we control
the producer (the I1 regime); it **sidesteps rather than solves** the real
inverse problem, which is exactly the I2 photo-cascade case where no producer
exists. This is the precise sense in which the retro's "we built a round-trip
validator, not yet an inference engine" (§0, §1 cap-1) is corroborated by the
field: the validated-by-literature part is the loop; the part with no analog is
the metadata shortcut, and it only works when we are the producer.

# 2. Dead-end verdicts (the core of the request)

| Dead-end (retro §3/§5) | Verdict | Why |
|------------------------|---------|-----|
| **(a) girih via hand-authored primitives** | **Confirmed dead** | Lu & Steinhardt (Science 2007): girih fields are generated by a finite 5-tile set (decagon, pentagon, hexagon, bowtie, rhombus) + a *published, finite, deterministic subdivision rule*; "A subdivision rule, combined with decagonal symmetry, is sufficient to construct perfect quasi-crystalline tilings." Hand-placing primitives is the genuine dead-end; the substitution-rule engine is the correct generative model. **Directly validates the bikar girih-substitution-rule-engine plan.** |
| **(b) derotation by shape-count cost** | **Refuted** | Arkin et al. (TPAMI 1991): the turning-function metric is "invariant under translation, rotation, and change-of-scale"; rotation is a shift of the turning function, and "the rotation angle θ* from the best matched turning functions represents the orientation difference … and can be used for orientation correction." Shape-count cost was abandoned for the right reason, but the *replacement already exists in our citations* — we should use TF orientation recovery, not invent one. |
| **(c) raster fold=1 as "honest observation"** | **Open-in-field** | No surveyed source treats raster-recovered symmetry fold as an honest-vs-threshold question. This is a qiyas-internal engineering call with no published analog (the retro already records the MSE-threshold fix as the resolution; prior art neither confirms nor refutes). Symmetry *detection* literature exists (Lee-Liu CMU 2008, surfaced but not load-bearing) but not the "fold=1 honest observation" framing. |
| **(d) clip/boolean-primitive medallion variants** | **Open-in-field** | No surfaced source addresses clip/boolean-primitive *construction* variants for medallion fields. The CSG literature uses booleans as the *generative grammar* (union/intersect/subtract), which is the opposite direction from "detect a clipped field"; no published analog for the iter-20/21 clip-probe regressions. Genuinely our territory. |
| **(e) I2 photo cascade** (rasterize→trace→re-derive, HSV seg, no producer) | **Open-in-field — and provably ill-posed** | The general inverse problem (arbitrary photo → DSL) is hard and the cited loops all operate on CAD shapes / 3D meshes / photoreal scenes, **never Islamic patterns**. The image-tracing step a photo needs is its own unsolved sub-problem (raster→vector references surfaced: USPTO 9036915, line-vectorization, image-tracing). **Foundational caveat:** the forward map is *many-to-one* — Kaplan (2005): "different tilings may give rise to the same star pattern under suitable choices of contact angle" — so there is **no single ground-truth construction** for a pattern image. The I2 inverse is inherently non-unique. |

# 3. Synthetic-GT training — Confirmed established

**Confirmed.** A generator emitting imagery *with* automatically-generated exact
labels is a standard CV technique for both training and benchmarking. CSGNet:
"we automatically created a large set of 2D and 3D CSG-based synthetic programs …
We sampled derivations of the … CSG grammar to create our synthetic dataset" —
the exact shape of "a DSL generator emits patterns with known DSL labels."
Gaidon-Lopez-Perronnin (IJCV 2018, *The Reasonable Effectiveness of Synthetic
Visual Data*): synthetic datasets "come with automatically generated ground
truth," treated as accepted practice. **This validates the retro §5
auto-generated-training-pipeline alternative** (bikar emits patterns w/ known GT
→ qiyas verifies at scale). Caveat from verification: CSGNet found synthetic-only
training insufficient — needed a supervised + RL two-stage; so a synthetic
pipeline is the *substrate*, not a complete solution.

# 4. The foundational caveat that reframes the whole project

**The inverse problem is ill-posed because the forward map is non-injective.**
Kaplan's polygons-in-contact forward generator is low-dimensional (a tiling + a
contact angle θ, optional δ) — which *validates* the premise that constructions
are parametric and recoverable in principle. But the same paper proves the map is
many-to-one: distinct (tiling, θ) pairs render the *identical* pattern. Therefore
**"recover *the* construction" is the wrong goal** — there is no single ground
truth. The principled target is an *equivalence class* (or a
minimum-description-length / canonical representative within it). This is the
deepest open question the research surfaced and it directly informs Phase 3's
"what does the loop loop-until?" question (retro §5) and the F2 identity-label
saga (a label is canonical-geometry, never the author's name — which the
2026-05-29 F2 doc independently rediscovered).

# 6. What NOT to overstate (verification killed 2 claims)

Verification **refuted** two over-broad claims, so the report does not assert
them: (1) that round-trip acceptance is robust to grammar-inexpressible inputs *in
general* (it isn't — InverseCSG's robustness is scoped); (2) that inverse
procedural modeling is flatly "solvable" for the general problem (it is not — the
general inverse is hard, and the many-to-one forward map makes it provably
non-unique). CSGNet's inference-time refinement also only optimizes *continuous
parameters with structure fixed* — it is not a full structure search. Keep these
in view so the project doesn't claim the literature solved more than it did.

# 7. Answer to the request's three questions

1. **Is the producer-checked round-trip consistent with prior art?** Yes,
   Confirmed (CSGNet/InverseCSG/VIGA) — except the metadata-as-source-of-truth
   shortcut, which is novel/Open-in-field and only valid while we are the producer.
2. **Are the dead-ends genuinely dead, or solved elsewhere?** (a) Confirmed dead
   → use substitution-rule. (b) Refuted → use turning-function orientation
   (already cited). (c), (d), (e) Open-in-field → genuine engineering territory,
   with (e) provably non-unique (Kaplan).
3. **Is synthetic-GT training established?** Yes, Confirmed (CSGNet, Gaidon IJCV
   2018) — validates the auto-generated-training-pipeline alternative.
